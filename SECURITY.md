# Security Policy

## Supported versions

Only `main` is supported. Older tags are not patched.

| Version       | Supported |
| ------------- | --------- |
| `main`        | ✅        |
| anything else | ❌        |

## Reporting a vulnerability

**Do not** open a public GitHub issue.

Email **security@kingdomcars.example** with:

- A clear description of the issue
- Reproduction steps or PoC
- Affected URL / version / commit
- Your name (optional) and contact

We will acknowledge within **2 business days** and aim to provide a remediation plan within **10 business days**.

If we accept the report, we'll coordinate a disclosure date with you. Please give us a reasonable embargo window before any public discussion.

## Out of scope

- Reports without a clear security impact (e.g. theoretical CSRF on idempotent GET endpoints)
- Findings exclusively from automated scanners with no exploit demonstrated
- Social-engineering attacks against staff

## Safe harbour

Researchers acting in good faith and following this policy will not be subject to legal action.
